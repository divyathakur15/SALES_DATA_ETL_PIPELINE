# 📊 Sales Data ETL Pipeline

A beginner-friendly but professional **ETL (Extract, Transform, Load) pipeline** for sales data, built with Python and MySQL. Includes data quality checks, a star-schema data warehouse, analytics SQL queries, and matplotlib visualizations.

---

## 🗂️ Project Structure

```
sales-data-etl-pipeline/
│
├── main.py                    # Pipeline entry point — run this!
├── generate_data.py           # One-time script to generate 1000 rows of sample data
├── requirements.txt
├── .gitignore
│
├── config/
│   └── .env.example           # Copy to .env and add your MySQL credentials
│
├── data/
│   ├── raw/
│   │   └── sales_data.csv     # Raw input data (generated by generate_data.py)
│   └── processed/             # Output: cleaned dimension & fact tables (CSVs)
│
├── src/
│   ├── extract/
│   │   └── extract.py         # Step 1: Read CSV, validate columns
│   ├── quality/
│   │   └── quality.py         # Step 2: Missing values, duplicates, type checks
│   ├── transform/
│   │   └── transform.py       # Step 3: Build star schema tables
│   ├── load/
│   │   └── load.py            # Step 4: Insert into MySQL (or save CSVs)
│   └── visualize/
│       └── visualize.py       # Charts: trends, regions, top products
│
├── sql/
│   ├── create_tables.sql      # Star schema DDL (CREATE TABLE statements)
│   └── analytics_queries.sql  # 7 analysis queries for MySQL
│
└── docs/
    └── charts/                # Generated PNG charts
```

## 🏗️ Database Design — Star Schema

```
         dim_date          dim_product
        ─────────         ────────────
        date_id  ◄──┐  ┌─► product_id
        full_date    │  │   product_name
        year         │  │   category
        month        │  │
        weekday      │  │
                     │  │
              fact_sales
             ────────────────
             order_id  (PK)
             date_id   (FK)  ──► dim_date
             product_id(FK)  ──► dim_product
             customer_id(FK) ──► dim_customer
             region
             quantity
             unit_price
             total_amount
             profit
             profit_margin_pct
             status
                  │
                  └──► dim_customer
                       ────────────
                       customer_id
                       customer_name
                       customer_email
                       customer_type
```

## ⚙️ ETL Pipeline Steps

| Step | File | What It Does |
|------|------|-------------|
| **Extract** | `src/extract/extract.py` | Reads CSV, validates required columns exist |
| **Quality** | `src/quality/quality.py` | Drops nulls, removes duplicates, validates numeric values |
| **Transform** | `src/transform/transform.py` | Builds `dim_product`, `dim_customer`, `dim_date`, `fact_sales` |
| **Load** | `src/load/load.py` | Inserts into MySQL using `INSERT IGNORE` (idempotent) |

## 🚀 How to Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Generate the sample dataset
```bash
python generate_data.py
# → Creates data/raw/sales_data.csv (1000 rows)
```

### 3. Run the ETL pipeline (CSV output — no MySQL needed)
```bash
python main.py
# → Outputs cleaned tables to data/processed/
```

### 4. Run with MySQL (optional)
```bash
# Copy and fill in your credentials
cp config/.env.example config/.env

# Then run
python main.py --mysql
# → Creates sales_dw database, loads all tables
```

### 5. Generate visualizations
```bash
python src/visualize/visualize.py
# → Saves 5 charts to docs/charts/
```

### 6. Run analytics queries (MySQL)
```sql
-- In MySQL Workbench or CLI:
source sql/analytics_queries.sql
```

- Designing a simple **star schema** data warehouse
